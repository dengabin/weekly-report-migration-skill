# 步骤 7：写回与格式约束

**本步目标**：仅将指定周次、指定成员的内容写入部门表，不破坏其它数据与超链接。

---

## 7.1 Agent 执行

```bash
python scripts/workflow/apply_migration.py --upload
```

（用户确认「仅本地」时去掉 `--upload`）

内部流程：format → ensure_week_column → plan → `patch_ksheet_zip.py` → 校验 → wps365-read `update`

## 7.2 写回技术约束

| 必须 | 禁止 |
|------|------|
| `.ksheet` 使用 `scripts/patch/patch_ksheet_zip.py` | openpyxl `save()` 写 `.ksheet` |
| 只改 `patch-plan` 内目标单元格 | 改其它周列、其它子表 |
| 保留 `customXml/item2.xml` 全部 hypersublink | 破坏含 📄 的组内周报链接列 |
| 从同行相邻历史周列复制单元格样式 `s` | 沿用占位符列样式导致不换行 |

## 7.3 内容约束

| 允许 | 禁止 |
|------|------|
| 列表符号、缩进、续行缩进 | 增删改词句 |
| 单元格换行样式 | 修改 URL 文本 |
| 覆盖**本周列**内容格 | 修改历史周列内容 |

## 7.4 下载件选择

优先使用 `.cache` 中含完整 `customXml`（`hypersublink` > 0）的 `.ksheet` 作为 patch 输入。

## 7.5 本步完成标志

- `.cache/dept-report-patched.ksheet` 已生成
- 若 `--upload`：云端返回成功及版本号
