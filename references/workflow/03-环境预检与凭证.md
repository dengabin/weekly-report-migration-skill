# 步骤 3：环境预检与凭证

**本步目标**：依赖就绪、WPS_SID 有效、部门表可读、子表已解析。

---

## 3.1 Agent 执行（用户不参与）

```bash
pip install -r requirements.txt
python scripts/workflow/preflight.py
```

读取 `.cache/preflight-report.json` 的 `status` 与 `checks`。

## 3.2 分支处理

| status | Agent 动作 |
|--------|------------|
| `ready` | 进入步骤 4 |
| `need_deps` | `pip install -r requirements.txt` → 重跑 preflight |
| `need_wps365_read` | 执行 `setup_wps365_read.py` 自动发现/安装 → 重跑 preflight；**禁止**让用户手填目录 |
| `need_wps_sid` | **见 §3.3** |
| `need_config` | 回步骤 2 |
| `dept_read_failed` | 凭证过期，同 §3.3 重新收集 sid |

## 3.3 缺 wps_sid（关键：不中断对话）

1. 向用户发送 [wps-sid-guide.md](../wps-sid-guide.md) 中的**完整获取步骤**（F12 → Application → Cookies → `wps_sid`）
2. AskQuestion 或明确提示：**「请在下一条消息直接粘贴 wps_sid」**
3. **等待**用户回复（不结束会话、不让用户跑命令）
4. 收到后执行：
   ```bash
   python scripts/workflow/setup_wps_sid.py "<粘贴值>"
   python scripts/workflow/preflight.py
   ```
5. `ready` 后**自动继续**步骤 4，无需用户再次发起

## 3.4 约束

- preflight `ready` **之前**禁止用 MCP `extract_yundoc_content` 读 `.ksheet`
- 小组 otl 可由 preflight 或 MCP 拉取到 `.cache/team-report.md`

- `resolved_sheet` 已解析（预检报告中的子表名）
