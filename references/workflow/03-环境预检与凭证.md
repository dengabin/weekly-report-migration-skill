# 步骤 3：环境预检与凭证

**本步目标**：依赖就绪、WPS_SID 有效、部门表可读、子表已解析。

---

## 3.1 Agent 执行（用户不参与）

```bash
pip install -r requirements.txt
python scripts/workflow/preflight.py
```

读取 `.cache/preflight-report.json` 的 `status` 与 `checks`。

## 3.2 分支处理

| status | Agent 动作 |
|--------|------------|
| `ready` | 进入步骤 4 |
| `need_deps` | `pip install -r requirements.txt` → 重跑 preflight |
| `need_wps365_read` | 检查 `vendor/wps365-read`；缺失时 `setup_wps365_read.py`；**禁止**让用户手填目录 |
| `need_wps_sid` | **见 §3.3** |
| `need_config` | 回步骤 2 |
| `need_dept_sheet` | **见 §3.5** |
| `dept_read_failed` | 凭证过期，同 §3.3 重新收集 sid |

## 3.3 缺 wps_sid（关键：不中断对话）

1. **简短说明**为何需要（读/写 `.ksheet` 须 wps365-read + Cookie；MCP 无法直接操作表格）。
2. **必须调用 `AskQuestion`**（禁止仅用正文结束回合）：
   - 标题示例：`周报迁移 — 需要 wps_sid`
   - 题干含获取步骤（见 [wps-sid-guide.md](../wps-sid-guide.md)）：F12 → Application → Cookies → `wps_sid` → 粘贴 Value
   - 选项示例：`我已粘贴 wps_sid（在回复中）` / `稍后提供`
3. **不得**在本回合以「当前阻塞」「等待你提供」作为最终状态结束；AskQuestion 发出后回合结束，用户回复后**立即续跑**。
4. 收到粘贴值后执行：
   ```bash
   python scripts/workflow/setup_wps_sid.py "<粘贴值>"
   python scripts/workflow/preflight.py
   ```
5. `ready` 后**自动继续**步骤 4～8（`run_preview` 等），无需用户再次发起。

## 3.4 约束

- preflight `ready` **之前**禁止用 MCP `extract_yundoc_content` 读 `.ksheet`
- 小组 otl 可由 preflight 或 MCP 拉取到 `.cache/team-report.md`

## 3.5 子表 tab 名不一致（`need_dept_sheet`，退出码 4）

KDC `get-file-content` 返回的子表列表**可能滞后**于云端实际 tab 名。preflight 在 `download` 后从 **ksheet zip 的 `workbook.xml`** 读取权威 `all_sheets`。

若 `config.dept_sheet.sheet_name` 已填写但不在该列表中：

1. **必须 AskQuestion**：列出 `checks.dept_sheets.all_sheets` 全部 tab 名，请用户选择正确名称（可能写错或部门表 tab 已更新）。
2. Agent 更新 `config.dept_sheet.sheet_name` 后重跑 `preflight.py`。
3. `ready` 后**自动续跑**后续步骤。

**禁止**在用户环境新建临时 `.py` 列出子表；使用 `scripts/plan/list_dept_sheets.py --input .cache/<部门表>.ksheet`。
