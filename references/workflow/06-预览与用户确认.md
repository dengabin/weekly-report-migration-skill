# 步骤 6：预览与用户确认

**本步目标**：向用户展示迁移计划，**获得写回授权**。

---

## 6.1 必须展示的信息

> **注意**：`run_preview.py` 在生成计划前可能调用 `ensure_week_column` 修改**本地** `.cache/dept-report.ksheet`（插入周列等），并非纯只读；云端部门表此时不变，写回须用户确认后 `apply_migration --upload`。

| 列 | 说明 |
|----|------|
| 子表 | `resolved_sheet` |
| 周次 | `config.week` |
| 成员 | 姓名 |
| 目标单元格 | 如 `C7` |
| 状态 | `ready` / 跳过原因 |
| 内容摘要 | 前 60–80 字 |

示例：

```
子表: <子表名> | 周次: YYYY-MM-DD | 共 N 人 ready
  <成员A>: <单元格> [ready] • 本周工作总结：...
  <成员B>: <单元格> [ready] • 本周工作总结：...
```

## 6.2 AskQuestion（强制）

- **必须调用 `AskQuestion`** 获得写回授权；禁止仅用正文问「是否确认写回」后结束回合。
- 缺确认**不是**任务结束；用户回复后**同一会话**进入步骤 7 或按用户选择取消。对话不中断原则见 [01-原则与用户边界.md](01-原则与用户边界.md) §1.3。

选项建议：

- **确认写回云端**
- **仅本地 patch，不上传**
- **取消**
- （可选）仅迁移指定成员

**未确认前禁止执行 `apply_migration.py --upload`。**

写回确认 AskQuestion 的题干**必须醒目展示** `resolved_sheet`（子表 tab 名）、`week`、ready 人数；若子表来自用户刚点选，须再次写明「将写入子表：XXX」，避免选错页误写它组。

## 6.3 用户可补充的指令

- 「只同步张三和李四」→ 过滤 `patch-plan` 中对应项
- 「先不上传」→ 步骤 7 不加 `--upload`

## 6.4 本步完成标志

- 用户已明确确认或取消
- 若确认，进入步骤 7
