# 步骤 6：预览与用户确认

**本步目标**：向用户展示迁移计划，**获得写回授权**。

---

## 6.1 必须展示的信息

| 列 | 说明 |
|----|------|
| 子表 | `resolved_sheet` |
| 周次 | `config.week` |
| 成员 | 姓名 |
| 目标单元格 | 如 `C7` |
| 状态 | `ready` / 跳过原因 |
| 内容摘要 | 前 60–80 字 |

示例：

```
子表: 应用研发-AI应用组 | 周次: 2026-07-02 | 共 16 人 ready
  邵田陆: C2 [ready] • 本周工作总结：...
  张晋涛: C7 [ready] • 本周工作总结：...
```

## 6.2 AskQuestion（强制）

选项建议：

- **确认写回云端**
- **仅本地 patch，不上传**
- **取消**
- （可选）仅迁移指定成员

**未确认前禁止执行 `apply_migration.py --upload`。**

## 6.3 用户可补充的指令

- 「只同步张三和李四」→ 过滤 `patch-plan` 中对应项
- 「先不上传」→ 步骤 7 不加 `--upload`

## 6.4 本步完成标志

- 用户已明确确认或取消
- 若确认，进入步骤 7
