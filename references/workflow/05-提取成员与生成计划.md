# 步骤 5：提取成员与生成计划

**本步目标**：从组内 otl 提取指定周每人内容，格式化，生成 `patch-plan.json`。

---

## 5.1 Agent 执行

推荐一键：

```bash
python scripts/workflow/run_preview.py
```

等价分步：

```bash
python scripts/extract/extract_otl_weekly.py --markdown .cache/team-report.md --output .cache/extracted.json [--week <config.week>]
python scripts/plan/build_config_from_extracted.py
python scripts/plan/resolve_team_name.py
python scripts/extract/format_otl_for_ksheet.py --input .cache/extracted.json --kdc-json .cache/dept-content.json --in-place
python scripts/plan/plan_sheet_patches.py --config config.json --dept-kdc-json .cache/dept-content.json --extracted .cache/extracted.json --output .cache/patch-plan.json
```

若 `resolve_team_name.py` **退出码 3**：停止本步，按 [team-name-resolution.md](../team-name-resolution.md) 询问用户组名，**禁止**继续 plan / 写回。

若 `config.members` 为空：

```bash
python scripts/plan/build_config_from_extracted.py
```

## 5.2 成员与组名

- **成员**：唯一真源为组内 otl 当前周次区块下的 `## 姓名`；禁止用用户口述名单覆盖
- **组名**：部门子表内多组并存时，用 `resolve_team_name.py` 根据姓名反推；**能唯一对应同一组则不问用户**；无法确定则 AskQuestion，**禁止猜组**
- 某人无内容：在计划中标注 `skipped`，预览时告知用户

## 5.3 格式化说明

`format_otl_for_ksheet.py` 仅做：

- 列表符 `-` → `•` / `◦` / `▪`
- 缩进、续行缩进（如无 `-` 前缀的表格行）
- 参考同行 E 列历史风格

**不改正文词句、URL**（`content_raw` 为语义真源）。

## 5.4 计划检查

- `resolved_sheet` 与 config 一致
- 每位成员 `status: ready` 且有 `cell`（如 `C7`）
- `patch-plan.content` 与 `extracted.content` 一致

## 5.5 本步完成标志

- `.cache/patch-plan.json` 已生成
- 预览摘要可展示（成员数、单元格、内容前 60 字）
