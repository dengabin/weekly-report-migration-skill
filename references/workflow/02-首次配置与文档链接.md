# 步骤 2：首次配置与文档链接

**本步目标**：确保 `config.json` 含有效文档链接；成员由 otl 自动填充。

---

## 2.1 何时执行本步

**前置条件**：

1. 用户已明确要**执行迁移**（如说「周报迁移」），而非仅「加载 skill」。
2. 若用户在**业务项目**工作：已配置 `.cursor/rules/weekly-report-migration.mdc`（通过「加载 skill + 路径」安装，见 [load-in-other-project.md](../load-in-other-project.md)）。

| 条件 | 动作 |
|------|------|
| 无 `config.json` | AskQuestion 收集链接 → Agent 写入 config |
| 有 config 但缺 `link_id` | 补问链接 |
| config 完整 | 快速核对链接仍有效，跳过问答 |

## 2.2 AskQuestion 收集项（按序，**一次只问一项**）

> **对话不中断**：本步所有索要（链接、组名）均须 **AskQuestion**，禁止仅用正文说「请在下一条消息提供」后结束回合；用户回复后**同一会话自动续跑**。详见 [01-原则与用户边界.md](01-原则与用户边界.md) §1.3。

> **⛔ 禁止合并询问**：组内链接与部门链接**必须分两次** AskQuestion，每次只索要**一个**链接（或一次 otl 正文）。禁止在同一题里写「请粘贴两个链接」「组内: … 部门: …」等合并格式。

| 轮次 | AskQuestion 主题 | 用户只需提供 |
|------|------------------|--------------|
| 第 1 次 | 组内周报链接 | **仅**组内 `.otl` 链接，或直接粘贴 otl Markdown 正文 |
| 第 2 次 | 部门周报链接 | **仅**部门 `.ksheet` 链接 |
| （按需）第 3 次 | 四级部门子表名 | 从列表中选一页 |
| （按需）第 4 次 | 页内组名 | 仅**多组分区**且 `ambiguous` 时：组标题行名称 |
| （按需）第 5 次 | otl 姓名核对 | 仅**平铺子表**且姓名对不上时：是否 otl 笔误 |

收到组内链接/正文后 Agent **先写入 config 或 `.cache/team-report.md`**，再**单独**发起第二次 AskQuestion 要部门链接。用户若某轮已在自由回复中提前给了下一项，Agent 收下即可，**下一轮仍不得合并索要**。

1. **组内周报** — 云文档链接（`.otl`）**或**用户直接粘贴的 Markdown（`# 日期` + `## 姓名`）  
   粘贴时 Agent 写入 `.cache/team-report.md`
2. **部门周报链接**（`.ksheet`）— **另起一次** AskQuestion，与组内分开  
   例：`https://365.kdocs.cn/l/DEPT_LINK_ID`
3. **四级部门子表名**（预检 `need_dept_sheet`、或无法**唯一**解析时）  
   从 Agent 给出的 **ksheet 真实**子表列表（`all_sheets`）中选择；**config 填写的 tab 名不在列表中时也必须问**（可能写错或已更新）
4. **页内组名**（仅多组分区、`resolve_team_name.py` 退出码 3 且 `ambiguous`）  
   部门表内多个组、且 otl 成员姓名无法唯一对应到同一组时，**必须**询问；能**唯一**对应则**不问**（见 [team-name-resolution.md](../team-name-resolution.md) §1.1）
5. **otl 姓名核对**（仅平铺子表、`not_found` + `layout: flat_sheet`）  
   otl `## 姓名` 与部门表登记名不一致时 AskQuestion；**勿**让用户在员工名之间选「组」

## 2.3 不要问用户

- ❌ 成员名单
- ❌ 周次（用户未指定时）
- ❌ WPS_SID（预检通过前不问）
- ❌ 组名（平铺子表；或多组分区且 otl 姓名已唯一对应同一组时）

## 2.4 写入 config 后

Agent 从模板或已有 config 合并用户链接，再执行：

```bash
python scripts/extract/extract_otl_weekly.py --markdown .cache/team-report.md --output .cache/extracted.json
python scripts/plan/build_config_from_extracted.py
python scripts/plan/resolve_team_name.py
```

`resolve_team_name.py` 先判平铺/多组：平铺且全员匹配 → `flat_sheet`（不问组名）；多组且唯一对应 → `resolved` 写入 `team_name`；否则退出码 3，按 `team-resolve.json` 分支 AskQuestion。

## 2.5 本步完成标志

- `config.json` 含 `team_report.link_id`、`dept_report.link_id`（或已有 `.cache/team-report.md`）
- `members[]` 已从 otl 生成
- `team_name` 已解析，或已明确需用户补充组名（暂停在 AskQuestion）

## 2.6 更新文档链接（用户主动更换 otl / ksheet）

用户说「**更新周报迁移的文档链接**」或迁移时说明「组内/部门链接换了」并贴新 URL 时：

1. **分两次 AskQuestion**（与首次相同）：先仅收组内 otl 链接或正文 → 写入 config → 再仅收部门 ksheet 链接
2. **禁止**在同一题合并两个链接
3. 更新后重跑 `build_config_from_extracted.py`（若改了组内）→ `preflight.py` → 自动续跑
4. **不必**删除整个 `config.json`；只覆盖对应 `link_id` 即可
