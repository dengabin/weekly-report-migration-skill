# 步骤 2：首次配置与文档链接

**本步目标**：确保 `config.json` 含正确文档链接；**不向用户索要成员名单**。

---

## 2.1 何时执行本步

- `config.json` 不存在
- 缺少 `team_report` / `dept_report` 链接
- 预检返回 `need_config`

若配置已完整（如 AI应用组示例），**快速核对**链接仍有效即可，跳过问答。

## 2.2 AskQuestion 收集顺序

1. **组内周报链接**（`.otl` 智能文档）  
   例：`https://365.kdocs.cn/l/cpqRAGyILoLO`
2. **部门周报链接**（`.ksheet` 表格）  
   例：`https://365.kdocs.cn/l/cqGvaEAyY8lG`
3. **四级部门子表名**（仅当后续预检无法 `resolved_sheet` 时再问）

## 2.3 不要问用户

- ❌ 成员有哪些人
- ❌ 本周是哪一周（除非用户主动说了相对周次以外的特殊需求；默认在步骤 4 自动解析）
- ❌ WPS_SID（留到步骤 3）

## 2.4 成员名单自动生成

配置写入后，在步骤 5 执行：

```bash
python scripts/extract/extract_otl_weekly.py --markdown .cache/team-report.md --output .cache/extracted.json
python scripts/plan/build_config_from_extracted.py
```

将 otl 中 `## 姓名` 写入 `config.members[]`。

## 2.5 本步完成标志

- `config.json` 含 `team_report.link_id`、`dept_report.link_id`、`dept_sheet` 等必要字段。
