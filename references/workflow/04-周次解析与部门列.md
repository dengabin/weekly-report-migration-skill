# 步骤 4：周次解析与部门列

**本步目标**：确定本次迁移的 `week`，并确认部门表有对应周列（无则插入）。

---

## 4.1 周次规则（详见 [week-resolution.md](../week-resolution.md)）

| 用户说法 | Agent 行为 |
|----------|------------|
| 未提及（只说「周报迁移」） | **不询问**；取组内 otl 最新且 ≤ 今天的 `# YYYY-MM-DD` 区块 |
| 本周 / 这周 | 同上 |
| 上周 / 上一周 | 按**系统当前日期**计算日历上周（Mon~Sun），在 otl 中匹配落在该区间内的 `# 日期` 区块 |
| 上上周 | 系统日历的上上周，同上 |
| `week 2026-07-02`、`6.26-7.2` 等 | 匹配对应区块，更新 `config.week` 与 `week_aliases` |

**组内周次与部门周次必须一致**，写入同一 `config.week`。

## 4.2 检查部门表周列

从 `.cache/dept-content.json` 目标子表表头查找与 `config.week` 匹配的列（`week_matches` 逻辑）。

### 列已存在

- 记录目标列索引，供步骤 5 `plan_sheet_patches` 使用
- 该列允许**覆盖**写入；其它列**禁止**改动

### 列不存在（常见：组内先写、部门表未开列）

1. 在**时间顺序正确**位置插入新列（参考相邻列表头，如 `7月2日\n周期：6.26-7.2`）
2. 复制相邻列样式；D 列保留「📄组内周报」链接结构（若有）
3. **仅**改动新列相关结构，其它列字节级不变
4. 使用 ZIP/XML 补丁，**禁止** openpyxl `save()` 写 `.ksheet`
5. 插入后重新拉取部门表 / 重跑 plan

## 4.3 更新配置

Agent 将解析结果写入 `config.json` 的 `week` 字段（用户不手改）。

## 4.4 本步完成标志

- `config.week` 已确定
- 部门表存在对应周列（已有或已插入）
