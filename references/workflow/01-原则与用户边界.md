# 步骤 1：原则与用户边界

**本步目标**：明确 Agent 与用户分工，建立全程约束。

---

## 1.0 安装 Skill ≠ 执行迁移

| 用户说 | Agent 行为 |
|--------|------------|
| 「加载 / 安装这个 skill」 | 仅确认 Skill 目录、可选复制到 `~/.cursor/skills/` → **结束** |
| 「周报迁移」等触发词 | 进入步骤 2～8 完整流程 |

**安装阶段禁止**：运行 `preflight.py`、AskQuestion 索要文档链接、`pip install`。

---

## 1.1 用户不需要做的事

| 禁止要求用户 | Agent 代替完成 |
|--------------|----------------|
| 运行 `python scripts/...` | Shell 执行全部脚本 |
| `pip install` | 自动安装 `requirements.txt` |
| 下载/上传云文档 | wps365-read `download` / `update` |
| 手改 `config.json` | Agent 读写配置 |
| 提供组成员名单 | 从组内 otl 自动解析 `## 姓名` |
| 未指定周次时选择「哪一周」 | 默认组内 otl 最新一期 |
| 在多组部门表中猜组名 | `resolve_team_name.py` 自动解析；无法确定时**必须**问用户 |

## 1.2 用户可能需要做的事

| 场景 | 用户动作 | Agent 动作 |
|------|----------|------------|
| 首次使用 | 分两次提供组内链接、部门链接（或粘贴组内 otl 正文） | **两次** AskQuestion 各收一项 → 写入 config |
| 部门表多组且无法从姓名确定组 | 在下一条消息说明**组标题行名称** | `resolve_team_name.py` 无法唯一确定时 AskQuestion；**禁止猜组** |
| 缺 `wps_sid` | 在 AskQuestion 中粘贴 Cookie 值 | 简短说明原因 → **必须 AskQuestion** → 收到后 `setup_wps_sid.py` → **同一会话自动续跑** |
| 写回前 | 在 AskQuestion 中点「确认」或「取消」 | 展示预览表，未确认不上传 |

## 1.3 对话不中断原则（强制）

- 缺 `wps_sid`、缺文档链接、缺组名、写回前确认 —— **都不是任务结束**，不得把回合当作「已完成」或「当前阻塞，请手动继续」。
- **必须**用 `AskQuestion` 向用户索要信息（禁止仅用正文说「请在下一条消息粘贴」而不弹问答）。
- 收集文档链接时：**组内与部门分两次 AskQuestion**，每次只问一个链接；**禁止**在同一题合并索要两个链接。
- AskQuestion 的选项/说明中可嵌入 [wps-sid-guide.md](../wps-sid-guide.md) 的获取步骤（F12 → Cookies → `wps_sid`）。
- 用户回答后 Agent **在同一对话内**执行 `setup_wps_sid.py` → 重跑 preflight → **自动续跑**后续步骤，无需用户再次说「周报迁移」。
- TodoWrite 中当前步骤保持 `in_progress`，直到凭证配置完成。
- 禁止说「请你本地运行 preflight 后再找我」「配置好后告诉我」。

## 1.4 本步完成标志

- **仅安装**：已告知用户说「周报迁移」即可开始，**本回合结束**。
- **已触发迁移**：Agent 已理解边界，进入步骤 2。
