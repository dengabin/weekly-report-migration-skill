# 周报迁移 Skill — 新手指南

将**组内智能文档周报**（`.otl`）里每位成员的内容，自动填入**部门周报表格**（`.ksheet`）对应单元格。

**你不需要会写代码、不需要开终端。** 在 Cursor 里对 Agent **说一句话**即可；Agent 自动安装依赖、读云文档、生成预览、写回表格。你只在对话里回答问题、最后说「确认」。

## 两种说法，别搞混

| 你说 | Agent 做什么 | 会不会要文档链接 |
|------|--------------|------------------|
| **加载这个 skill：**`d:\...\report-migration`（在业务项目里） | 自动配置 `.cursor/rules/`，简述能力 | **不会** |
| **加载 skill**（已打开 Skill 开发仓库） | 确认就绪 | **不会** |
| **周报迁移** / 填部门周报 / 同步周报 | 完整迁移流程：预检 → 提取 → 预览 → 写回 | 首次会要（见下） |

> 准备好迁移了，再说 **「周报迁移」**。

## 迁移时：你要提供什么、Agent 会问什么

### 你通常要提供的（仅首次或你主动更换时）

| 内容 | 怎么说 / 怎么给 | 频率 |
|------|-----------------|------|
| 组内周报 | 粘贴 **otl 云文档链接**，或直接**粘贴周报正文**（见下） | 首次 |
| 部门周报 | 粘贴 **ksheet 云文档链接** | 首次 |
| `wps_sid` | 按 Agent 图文步骤从浏览器 Cookie 复制粘贴 | 首次或过期时 |
| 部门子表名 | 从 Agent 列出的子表里选一个 | 少数情况 |
| 小组组名 | 告诉 Agent 部门表里你们组的**组标题行文字** | 仅当部门表多组且无法从姓名自动判断时 |

### Agent 不会向你要的

| 不会问 | 原因 |
|--------|------|
| 组成员名单 | 从组内 otl 的 `## 姓名` 自动读取 |
| 「要迁移哪一周？」（你没提周次时） | 默认取组内 otl **最新一期**（日期 ≤ 今天） |
| 小组组名（多数情况） | otl 里姓名能在部门表唯一对应到同一组时自动识别 |

### 组内周报：链接 or 直接粘贴正文

两种方式任选其一：

**方式 1 — 链接（推荐，可重复使用）**

```
https://365.kdocs.cn/l/你的组内otl链接ID
```

**方式 2 — 直接粘贴 Markdown 正文**

在对话里粘贴组内 otl 内容，格式须包含：

```markdown
# 2026-07-02

## 张三
- 本周工作总结：...

## 李四
- 本周工作总结：...
```

即：`# 日期` 作为周次标题，其下用 `## 姓名` 分成员。Agent 会保存并解析，**不必同时再给链接**。

## 指定周次：怎么说才没有歧义

不说周次时，Agent 取组内 otl **最新一期**（不额外问你）。

| 你想迁移 | 在对话里说 | 实际含义 |
|----------|------------|----------|
| 默认（最新一周） | `周报迁移` | otl 中日期 ≤ 今天的最新 `# 日期` 区块 |
| **日历上周** | `迁移上周周报` / `周报迁移，上周` | 按**今天系统日期**算上一个自然周（周一～周日），再在 otl 里找落在该周的 `# 日期` 区块 |
| 日历上上周 | `周报迁移，上上周` | 再往前一个自然周 |
| 指定某一天 | `周报迁移，week 是 2026-07-02` | 精确匹配该日期区块 |
| 按部门表表头 | `周报迁移，7月2日那一列` | 匹配部门表对应周列 |

**重要 — 「上周」不是 otl 里的上一期**

举例：今天系统日期是 **2026-07-08**，组内 otl 有 `# 2026-07-02` 和 `# 2026-06-25` 两期：

- 说 **「上周」** → 迁移 **`# 2026-07-02`**（日历上周 6/29～7/5 落在这一期）
- **不是** otl 里排在后面的 `# 2026-06-25`

## 使用前准备

1. **Cursor**（已登录）
2. **Python 3.10+**（本机有即可，不用自己运行）
3. 对组内 **otl**、部门 **ksheet** 有读/写权限
4. 本仓库已 `git clone` 到本机（内置 `vendor/wps365-read`，无需另装）
5. **业务项目**已让 Agent 执行过「加载 skill + 路径」（见下一步）；或你正在打开 Skill 开发仓库自测

## 第一步：在业务项目里启用

在**你的业务项目**打开 Cursor，粘贴 Skill 目录并说：

```
加载这个 skill：d:\branch\skills\report-migration
```

Agent 会 Read 安装文档并**自动**在当前项目创建 `.cursor/rules/weekly-report-migration.mdc`（仅此一个文件，无需手跑命令）。

详见 [references/load-in-other-project.md](references/load-in-other-project.md)。

### 方式 A：贴路径 +「加载 skill」（业务项目，推荐）

Agent 自动配置指针 Rule；迁移时说「**周报迁移**」。

### 方式 B：打开 Skill 开发仓库当工作区

改 Skill 代码、自测；仅在该仓库内生效。说「加载 skill」确认即可，**不必**跑 `install_to_project.ps1`。

### 方式 C：手动安装（Agent 不可用时）

```powershell
powershell -ExecutionPolicy Bypass -File "<SKILL_ROOT>/scripts/workflow/install_to_project.ps1" -SkillRoot "<SKILL_ROOT>" -TargetProject "D:\your-app"
```

详见 [references/install-project-rule.md](references/install-project-rule.md)。

## 第二步：完整迁移流程（第一次）

### 1. 触发

在 Cursor 对话中说：

```
周报迁移
```

Agent 会立刻在侧边 **Todo** 面板创建 **步骤1～步骤8** 共八项（与 workflow 文档一一对应），每完成一步打勾，方便你看清进度。**这是强制流程展示，不会省略。**

### 2. 提供文档（仅首次无 config 时）

Agent **分两次**询问，你每次**只粘贴一个**链接即可：

1. 组内 otl 链接（或粘贴周报正文）— **第一次**  
2. 部门 ksheet 链接 — **收到组内后再问第二次**  
3. （少数情况）从子表列表里选你们四级部门那一页 — **另起一次**

### 3. 凭证（仅首次或过期）

Agent 引导你从浏览器复制 `wps_sid`，你**回到对话粘贴**，不用跑任何命令。  
详见 [references/wps-sid-guide.md](references/wps-sid-guide.md)。

### 4. 预览 → 确认 → 写回

Agent 展示：**周次、子表、成员、目标单元格、内容摘要**。

核对无误后回复：

```
确认
```

**未确认前不会上传。** 写回后打开部门表，**Ctrl+F5** 强刷查看。

### 对话示例

```
你：  周报迁移

Agent：请提供组内周报链接（或直接粘贴 otl 正文）
你：  https://365.kdocs.cn/l/TEAM_LINK_ID

Agent：请提供部门周报链接
你：  https://365.kdocs.cn/l/DEPT_LINK_ID

Agent：（若部门表多组且无法自动识别）请提供你们组在部门表里的组名
你：  示例小组名

Agent：（若缺凭证）请粘贴 wps_sid
你：  （粘贴 Cookie 值）

Agent：预览 16 人 → 周次 2026-07-02，是否确认写回？
你：  确认

Agent：已写回，请 Ctrl+F5 刷新部门表
```

## 重要说明

### 写回范围

- **只改**指定周的**内容列**；历史周、其它子表、📄 链接列不动  
- 仅调整列表符、缩进等排版，**不改正文词句**

### 部门表里有多个组时

- otl 里**有成员姓名**，且姓名在部门表能**唯一对应到同一组** → Agent **不问组名**，直接迁移  
- 无法判断属于哪一组 → Agent **会问你组名**，**不会**随便写到别的组

### 本地缓存与 config

写回成功后默认清空 `.cache/`（中间文件）。**下次迁移会重新从云端拉**组内 otl 与部门表，不靠上次 `.cache`。

`config.json`、`wps_sid` 写在 **Skill 目录（SKILL_ROOT）** 内，不在业务项目根目录。会保留文档链接与凭证；**组名每次迁移会自动按最新部门表重算**。若换了部门文档，说「**更新周报迁移的文档链接**」。

### `wps_sid` 会过期

按 Agent 提示重新粘贴即可，无需重装 Skill。

## 常见问题

| 现象 | 怎么办 |
|------|--------|
| 不知道从哪开始 | 业务项目：`加载这个 skill：<路径>` → `周报迁移` |
| 说「加载」却开始要链接 | 回复：「我只是加载 skill，还没要迁移」 |
| 设置里看不到 / 没有 `/` | **正常**；本 Skill 只用指针 Rule，说「周报迁移」或 @weekly-report-migration |
| 「上周」迁错了周 | 确认你说的是「日历上周」；otl 上一期 ≠ 日历上周（见上文） |
| 部门调整了组名 / 组织 | 一般不用重做：下次迁移会重拉部门表并自动更新 `team_name`；无法判断时才再问组名 |
| 换了部门文档或子表 | 说「更新周报迁移的文档链接」，或删除 `config.json` 后重来 |
| 部门表看不到更新 | **Ctrl+F5** 强刷 |
| Agent 让你跑 python | 回复：`请按 Skill 要求你自动执行` |

## 组内 otl 格式要求

迁移前请确认组内 otl 结构如下：

```markdown
# 2026-07-02

## 王亮
- 本周工作总结：...
- 下周工作计划：...

## 李四
...
```

- 每周一个 `# YYYY-MM-DD`（或 Agent 能识别的日期标题）  
- 每人一个 `## 姓名`  
- 部门表按「组名 + 姓名 + 周次列」定位单元格（Agent 自动匹配）
